# 需求文档

## 1. 应用概述

**应用名称**：Huky个人主页

**应用描述**：一个简约风格的个人主页网站，用于向客户和潜在合作伙伴展示Huky的个人信息，并提供数字分身聊天功能，访客可通过AI对话了解Huky的专业领域和观点。

## 2. 用户与使用场景

**目标用户**：
- 客户
- 潜在合作伙伴

**核心使用场景**：
- 访客浏览个人信息，了解Huky的职业背景和专业方向
- 访客通过数字分身聊天区提问，获取关于教育产品、职业规划等问题的解答

## 3. 页面结构与功能说明

### 3.1 页面结构

```
个人主页
├── 个人信息展示区
│   ├── 头像
│   ├── 名字和一句话介绍
│   └── 个人信息详情
└── 数字分身聊天区
    ├── 聊天对话框
    └── 输入区
```

### 3.2 个人信息展示区

**3.2.1 头像**
- 展示Huky的头像图片

**3.2.2 名字和一句话介绍**
- 名字：Huky
- 一句话介绍：教育产品专家，史哲人类爱好者

**3.2.3 个人信息详情**
- 我现在主要在做：教育产品设计与营销
- 我的兴趣：产品设计与营销
- 我一个比较有记忆点的特点：就是没有

### 3.3 数字分身聊天区

**3.3.1 聊天对话框**
- 展示访客与数字分身的对话历史
- 数字分身回复以Huky的身份和视角呈现

**3.3.2 输入区**
- 访客输入问题
- 发送按钮，点击后将问题发送给数字分身

**3.3.3 AI对话能力**
- 接入AI大模型，处理访客提问
- AI基于以下背景信息回答问题：
  - 职业或身份：教育产品专家，史哲人爱好者
  - 最近在做的事：教育产品设计
  - 擅长或关心的方向：人
  - 常见问题：学什么，未来怎么样，怎么规划

## 4. 业务规则与逻辑

### 4.1 数字分身对话规则
- 访客提问后，AI大模型基于Huky的背景信息生成回复
- 回复内容需符合Huky的身份定位和专业领域
- 对话历史在当前会话中保留，刷新页面后清空

### 4.2 响应式适配
- 页面需适配手机端，确保在移动设备上正常展示和使用

### 4.3 视觉风格
- 整体风格：简约
- 配色方案：蓝橙配色

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 访客输入空白问题 | 提示「请输入问题」，不发送请求 |
| AI大模型响应超时 | 提示「回复超时，请稍后重试」 |
| AI大模型返回错误 | 提示「回复失败，请稍后重试」 |
| 头像图片加载失败 | 显示默认占位图 |

## 6. 验收标准

1. 访客打开个人主页，看到头像、名字「Huky」、一句话介绍「教育产品专家，史哲人类爱好者」
2. 访客查看个人信息详情，看到「我现在主要在做：教育产品设计与营销」「我的兴趣：产品设计与营销」「我一个比较有记忆点的特点：就是没有」
3. 访客在数字分身聊天区输入问题「学什么比较好」
4. 访客点击发送按钮
5. 数字分身返回基于Huky身份和视角的回复内容
6. 访客在手机端打开页面，页面正常展示且可正常使用聊天功能
